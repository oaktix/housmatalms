## Long-Form Draft

### Title
Product Manager Skills v0.65: You Asked, We Listened

### Subtitle
Comprehensive PM-first setup guidance for installing and integrating skills across chat, CLI, automation, and agent frameworks

### Article Body

v0.65 is a documentation and onboarding release driven directly by user feedback.

The core request was clear:
- Make setup less ambiguous
- Make platform paths explicit
- Make guidance friendlier for PMs who are not programmers

So we focused this release on practical, tool-specific instructions that help people get to first value faster.

What changed:

1. **Beginner-first onboarding**
   - Added `docs/Using PM Skills 101.md` as the canonical "start here" guide
   - Expanded real-world setup patterns across major PM-adjacent tooling

2. **Platform decision layer**
   - Added `docs/Platform Guides for PMs.md` as a simple chooser page
   - Added clear routing by comfort level and workflow type

3. **Claude slash command operationalization**
   - Added `docs/Using PM Skills with Slash Commands 101.md`
   - Includes ready-to-copy command patterns like `/pm-story`, `/pm-prd`, `/pm-probe`, and `/pm-prioritize`

4. **PM-friendly platform guides**
   - Added practical guides for:
     - Claude Code
     - Claude Desktop
     - Claude Cowork
     - ChatGPT Desktop
     - OpenClaw
     - n8n
     - LangFlow
     - Python agents

5. **Onboarding path refresh**
   - Updated `START_HERE.md` so users can pick: chat-first, terminal-first, automation-first, or experimental stack

The outcome:
- Less setup friction
- Faster time to first useful output
- Better continuity between non-technical PMs and technical implementation partners

If v0.6 was about navigation and command wrappers, v0.65 is about making the whole library easier to adopt in the real world.
