# Using PM Skills with Bolt

If you are new to PM Skills, start with [`Using PM Skills 101.md`](Using%20PM%20Skills%20101.md).

Bolt is best when you want rapid prototyping, but skills make prototypes decision-grade instead of vibe-grade.

## Best For

- PMs validating product ideas quickly
- Prototype-first discovery loops
- Framing problems before generating UI/code

## 10-Minute Setup

1. Start a new Bolt project.
2. Before asking it to build, paste one PM skill.
3. Start with `problem-statement` or `pol-probe`.

Starter pattern:

```text
Before generating any code, apply this PM skill:
[Paste skills/problem-statement/SKILL.md]

Problem context: [your problem]
Ask clarifying questions first.
```

## First Win Prompt

```text
Apply the User Story skill below to define what we're building,
then generate a prototype that satisfies story #1.

[Paste skills/user-story/SKILL.md]
Feature: self-serve seat management for enterprise admins.
```

## Best Skills for Bolt

- `problem-statement`
- `pol-probe`
- `user-story`
- `proto-persona`

## Common Pitfalls

- Jumping straight to "build me X" with no framing.
- Treating first prototype output as validated strategy.
- Skipping acceptance criteria.
